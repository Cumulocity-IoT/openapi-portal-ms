Gainsight Sync Microservice API (0.2.2)

Download OpenAPI specification:

A NestJS microservice that periodically pulls Gainsight PX telemetry (custom events, session events, page views, active-user metrics) from configured tenant domains, caches the data in memory per tenant using a date-sorted array with binary-search range queries, and exposes REST endpoints consumed by dashboards such as Grafana.

All data endpoints require Authorization: Basic <base64(user:password)> and a tenantId query parameter. Public endpoints: /health, /lastRun, /llms.txt, /llms-full.txt.

Interactive docs: https://gainsight.eu-latest.cumulocity.com/apps/gainsight-c8y-openapi/index.html

end
required
string

End of the time range (ISO 8601 string or epoch milliseconds).

tenantId
required
string

Tenant identifier used to scope the cache query.

filter
string

filtrex filter expression applied to mapped user objects. Strings must be double-quoted.

Examples

  • country == "Germany" — users from Germany
  • platform == "desktop" and browser == "Chrome" — desktop Chrome users
  • device in ("mobile", "tablet") — mobile and tablet users only
  • roles != "guest" — exclude guest users
  • domain ~= "siemens" — users whose email domain contains "siemens"

string

Comma-separated user fields to return. Valid values: id, roles, country, sessionId, language, platform, browser, device, domain. Omit to return all fields.

Examples

  • id,country,roles — identity, location, and role breakdown
  • id,platform,browser,device — device and browser profile only
  • id,domain,language — domain and language breakdown
orderBy
string

Field and optional direction to sort results by, in the format field or field:asc / field:desc. Direction defaults to asc when omitted. Valid field values match those listed in fields.

Examples

  • country:asc — alphabetical by country
  • domain:asc — alphabetical by domain
  • language:asc — alphabetical by language
  • platform:desc — reverse-alphabetical by platform

v2/pageViews

Filter examples

  • All dashboards of group 123: hash ~= "group/123/dashboard"
  • All dashboards of device 456: hash ~= "device/456/dashboard"
  • Any dashboard across all groups or devices: hash ~= "(group|device)/[0-9]+/dashboard"
  • Specific user type: userType == "USER"
  • Multiple paths: path in ("/home", "/settings")

v2/pageViewCounts

Filters page-views using a filtrex expression (same syntax as /v2/pageViews), then groups the matching records by the groupBy field and counts occurrences. Results are always returned sorted by count descending so the most-visited item is first.

Typical use cases

  • Most popular dashboards within a group: filter=hash ~= "group/123/dashboard"&groupBy=hash&limit=10
  • Most popular dashboards within a device: filter=hash ~= "device/456/dashboard"&groupBy=hash&limit=10
  • Page-type popularity (collapse specific IDs into patterns): groupBy=hash&pathMask=true
  • Most active accounts: groupBy=accountId&limit=5
Authorizations:
basic
query Parameters
start
required
string

Start of the time range (ISO 8601 string or epoch milliseconds).

end
required
string

End of the time range (ISO 8601 string or epoch milliseconds).

tenantId
required
string

Tenant identifier used to scope the cache query.

filter
string

filtrex filter expression to scope page-view records before aggregation. Strings must be double-quoted. Use ~= for regex matching.

Examples

  • hash ~= "group/123/dashboard" — dashboards inside group 123
  • hash ~= "device/456/dashboard" — dashboards inside device 456
  • accountId == "acc-001" — views from a specific account
groupBy
string

Page-view field to aggregate by. Defaults to hash. Valid values: id, identifyId, sessionId, date, scheme, host, path, queryString, hash, remoteHost, referrer, pageTitle, propertyKey, eventType, userType, accountId.

pathMask
string
Enum: "true" "false"

When set to "true", all numeric segments in the grouped value are replaced with * before counting. This collapses individual IDs into page-type patterns. Example: #/group/123/dashboard/7 → #/group/*/dashboard/*.

limit
string

Maximum number of results to return. Applies after sorting, so you always get the top-N by count. Omit to return all groups.
